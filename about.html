Dovednosti

  • HTML 5

    Snažím se dodržovat sémantiku struktury HTML tagů pro SEO a Google vyhledávač. Používám správný typ elementu pro daný typ obsahu. Vyvaruji se nadměrného používání DIV elementů. Pro classy se snažím vycházet z BEM (Block element modifier) pojmenovávací konvence. Naučil jsem se používat picture element, pro různé druhy rozlišení, zařízení, druhů formátů a následných fallbacků.

  • Css 3

    Základní stylování dokumentu. Zaměřuji se na používání flexboxu a responzivního layoutu. Ve spolupráci se Sass je celkem jednoduché vytvořit celkovou podobu stránky. V poslední době jsem se zaměřil na pokročilejší funkce jako "animate" , "transition","transform".

  • Sass

    Neuvěřitelně efektivní věc. Nejen při psaní kódu, ale i při přípravě celkové struktury a přípravě a údržbě assetů pro další projekty. Používám tyto jeho funkce. "nesting", "@mixiny", "@placeholdery", cykly a proměnné. Diky funkci "@use" a "@forward" si můžu připravit konkretní sadu kódu na různá místa a později je připojit do hlavního SCSS kódu. Vytvářím si vlastní @funkce. Nepoužívám VSCode extension, ale přímo Sass terminal s Dart Sassem.

  • Javascript

    Práce s poly a proměnými. Targetování elementů v HTML DOM a přidávání nebo odebírání atributů. Získávání hodnot z formulářů a jejich vyhodnocení, uložení a použití na jiném místě. Používání podmíněných funkcí. Přidávání nových CSS vlastností.

  • Jquery

    Jen zběžné seznámení. Většinu funkcí teď nahradí CSS animace, nebo nové funkce v CSS. Třeba se to na něco menšího může hodit. Toto byla vstupní brána pro Javascript a díky tomu jsem mohl Js rychleji pochopit.

  • Bootsrap

    Začínám implementovat tento framework do svých projektů. Zejména Grid systém a další předpřipravené složité prvky, jako třeba Carousel. Dříve jsem byl spíše odpůrcem, protože jsem si myslel, že si všechno můžu napsat sám, ale zde můžu jednoduchým způsobem přidávat již hotové věci, které drobným zásahem v CSS můžu přetvořit k požadovanému stavu.

  • Git

    Umím inicializovat verzovací systém pro projekt. Následně commitovat a pushovat do úložistě. Vytvořil jsem si privátní úložiště na Githubu a zároveň jsem si zprovoznil Gitlab úložiště na svém domácím Proxmox serveru. Na obě úložiště odesílám změny v projektu. Zkouším vytvářet větve pro různé verze projektu. Ještě se potřebuji naučit jak používat Git v týmu.

  • IT Admin

    Operační systémy: Windows a Linux. Kancelářské systémy: Office a Libre Office. Virtualizační platformy: Virtual Box a Proxmox. V případě Linuxu jsem schopen nainstalovat, spustit a udržovat webservrové aplikace - Apache,Nginx,php. Nastavit load balancer, reverzní proxy atd. Zabezpečit linux server pomocí SSH a přehodit ho na jiný než defaultní 22 port a k tomu upravit nastavení Firewallu. Mám také základní znalosti CCNA certifikace a ovládání IT network periferií, switche a routery.

  • Gulp a npm

    Naučil jsem se používat npm . Společně s tím se mi do ruky dostal automatizovaný modulový system Gulp. Využívám moduly pro převod obrázků na webp formát, minifikaci a autoprefixaci css souborů. Vytvořil jsem si nějaké scripty, které mi pomáhají při zakládáni nových projektů a šetří mi tím čas.

Další Techniky

  • Používání variables fontů* a self hosting přes @font-face atribut. Převedení fontů na kompresní formát woff2 a vytvoření fallbacku ttf pro případ nekompatibility.

    *(Je potřeba si vypočítat co má větší efektivitu. Jestli budu používat jen dvě váhy fontu nebo celou sadu.)

  • Technika srcset pro používání retina displejů a funkci source media pro nahravání obrázků při různých rozlišení. tzv. "art direction".

  • Program Figma pro práci s navrženým wireframem.

  • Program Gimp pro práci s obrázky.

  • Lazy loading pro níže položené obrázky a přidám do HTML informaci o velikosti obrázku aby si ihned rezervoval místo v toku dokumentu a později tam nedoskakoval a ničil nastavený layout stránky. Zároveň přidávám barevný fallback pro obrázky, které se nenačtou.
